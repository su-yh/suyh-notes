步骤概览
步骤	执行节点	操作内容
1	node-1	生成CA证书和节点证书（含所有节点IP的SAN）
2	node-1	创建Elasticsearch Keystore并添加证书密码
3	node-1	更新 elasticsearch.yml 安全配置（先删除旧明文密码）
4	node-1	分发证书和配置到 node-2、node-3
5	node-2、node-3	重复步骤2和3（在本机创建Keystore并配置）
6	所有节点依次	启动Elasticsearch服务（先node-1，再其他）
7	任一节点（如node-1）	初始化内置用户密码（交互式设置）
8	node-1	验证集群安全状态




编号	用途	密码	长度	备注
P1	CA 证书密码（elastic-stack-ca.p12）	7dK!x@9Mq#2Lp$5R&vWz	20	用于生成 CA 时设置
P2	节点证书密码（elastic-certificates.p12）	aB3#eF8$gH2@jK5^mN7	18	用于生成节点证书时设置，也是 keystore 中要存储的密码
P3	Elasticsearch keystore 主密码（可选，但建议设置）	Qw!6Rt@9Yx#2Uv$4Pz	18	用于 elasticsearch-keystore create -p 时保护 keystore 文件本身
P4	内置 elastic 用户密码（后续 setup-passwords 使用）	El@st1c_S3cur3_P@ss!2026	26	集群管理员密码，建议格外复杂








生成 CA 证书
./bin/elasticsearch-certutil ca \
  --out config/certs/elastic-stack-ca.p12 \
  --pass '7dK!x@9Mq#2Lp$5R&vWz' \
  --days 3650

创建包含所有节点 SAN 的实例配置
cat > config/certs/instances.yml << 'EOF'
instances:
  - name: "node-1"
    ip: ["172.31.3.155"]
  - name: "node-2"
    ip: ["172.31.3.156"]
  - name: "node-3"
    ip: ["172.31.3.157"]
EOF

生成节点证书（每个节点都有一个独立身份，但这里用一张通用证书包含所有IP）
./bin/elasticsearch-certutil cert \
  --ca config/certs/elastic-stack-ca.p12 \
  --ca-pass '7dK!x@9Mq#2Lp$5R&vWz' \
  --out config/certs/elastic-certificates.p12 \
  --pass 'aB3#eF8$gH2@jK5^mN7' \
  --ip 172.31.3.155,172.31.3.156,172.31.3.157


# 验证证书是否包含 SAN
openssl pkcs12 -in config/certs/elastic-certificates.p12 -nokeys | openssl x509 -noout -text | grep -A 5 "Subject Alternative Name"
# 输入密码：aB3#eF8$gH2@jK5^mN7


导出 CA 公钥（供 Kibana、Spring Boot 等客户端使用），生成文件：ca.crt
openssl pkcs12 -in config/certs/elastic-stack-ca.p12 \
  -out config/certs/ca.crt -nokeys 
# 输入密码：7dK!x@9Mq#2Lp$5R&vWz



# 创建 keystore（首次需要设置密码）
./bin/elasticsearch-keystore create -p
# 输入新密码（如 Qw!6Rt@9Yx#2Uv$4Pz）

# 执行前先清空旧错误配置
./bin/elasticsearch-keystore remove xpack.security.transport.ssl.keystore.secure_password
# 输入：Qw!6Rt@9Yx#2Uv$4Pz
./bin/elasticsearch-keystore remove xpack.security.transport.ssl.truststore.secure_password
# 输入：Qw!6Rt@9Yx#2Uv$4Pz
./bin/elasticsearch-keystore remove xpack.security.http.ssl.keystore.secure_password
# 输入：Qw!6Rt@9Yx#2Uv$4Pz

# ========== transport 集群内部通信 ==========
# transport keystore：节点证书 elastic-certificates.p12 → P2

# 输入：aB3#eF8$gH2@jK5^mN7

# 输入：Qw!6Rt@9Yx#2Uv$4Pz
# 输入: 7dK!x@9Mq#2Lp$5R&vWz
