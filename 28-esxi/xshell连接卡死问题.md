# 方案1：更换为 e1000e 网卡（彻底解决 SSH 卡死问题）
### 操作步骤
1.  **关闭虚拟机**
    在 ESXi 6.5 的 Web 控制台中，找到你的 Ubuntu 模板虚拟机（如 `ubuntu_template_esxi`），点击 **关机 (Power Off)**，确保虚拟机处于完全断电状态。

2.  **进入虚拟机设置界面**
    右键点击已关机的虚拟机，在弹出的菜单中选择 **编辑设置 (Edit Settings)**。

3.  **删除原有 VMXNET3 网卡**
    在“虚拟硬件”标签页中，找到“网络适配器 1”，点击其右侧的 **移除 (Remove)** 按钮。

4.  **添加新的 e1000e 网卡**
    点击界面底部的 **添加其他设备 (Add other device)**，在下拉菜单中选择 **网络适配器 (Network Adapter)**。
    在新添加的“网络适配器 1”配置项中，找到 **适配器类型 (Adapter type)**，点击下拉菜单，选择 **E1000e**。
    确保“连接”和“打开电源时连接”两个选项都已勾选。

5.  **保存设置并启动虚拟机**
    点击 **保存 (Save)** 按钮退出设置界面。
    在虚拟机列表中，点击 **电源 (Power On)** 启动虚拟机。

6.  **验证连接**
    启动后，用 Xshell 等工具尝试 SSH 连接。如果能正常连接且不再卡死，说明问题已解决。

---

### 关键说明
- **原理**：ESXi 6.5 默认的 `VMXNET3` 网卡驱动与新版 Ubuntu 内核存在兼容性问题，而 `e1000e` 是一个更通用、更稳定的模拟网卡，对 Linux 的支持非常成熟。
- **后续模板化**：将这台虚拟机保持为关机状态，以后所有从它克隆出来的新虚拟机都会自动继承 `e1000e` 网卡配置，无需重复操作。
